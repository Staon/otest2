Enter an assertion.

The assertion is opened and closed similarly to states, cases and suites. While the assertion is opened some messages may be reported by the reportAssertionMessage() method. The assertion is closed by the method leaveAssert().

Parameters
context_the OTest2 context
condition_result of the assertion (false means failed assertion)
message_Initial assertion message. Usually it's shown at the same line as the rest of assertion data. More messages may be added by the method reportAssertionMessage().
file_name of the source file
lineno_line number in the source file

Implemented in OTest2::ReporterTee, OTest2::ReporterJUnit, and OTest2::ReporterConsole.

Enter an error report.

This method is used to report errors happened in the testing framework. Errors mean failure of the test. However, they cannot be located in the user code (no filename, no line number). A good example is a not handled exception caught by the framework outside the test case.

The method works as well as the enterAssert() method with failed condition and with no location info. The opened error state is closed by the method leaveAssert().

Parameters
context_the context
message_initial error message. The message is usually shown at the same line as the rest of the assertion reporting. More message may be added by the method reportAssertionMessage().

Implemented in OTest2::ReporterTee, OTest2::ReporterJUnit, and OTest2::ReporterConsole.
